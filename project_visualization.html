Project Type: ML Pipeline Domain: Software Quality Dataset: NASA PROMISE (KC1)

Goal (Plain Explanation)

Predict which code modules are likely to contain defects using static software metrics. This helps teams prioritize testing, reviews, and refactoring.

High risk modules → test first → fewer bugs.

Project Story in One Line

Decision Support System (DSS)

DSS transforms model predictions into a priority queue and risk report so managers can allocate testing resources.

  • Sort modules by predicted defect probability
  • Trigger alerts for high‑risk modules
  • Summaries in reports/

How to Run & Test

Run Pipeline

pip install -r requirements.txt
python main.py

Launch Dashboard

streamlit run dashboard/app.py

Check outputs in reports/ and data/processed/.

Limitations

Predictions rely on static metrics, not runtime behavior. The dataset is imbalanced (defects are ~15%), so recall is prioritized.

Future Extensions

Add more datasets (JM1, KC2), apply advanced balancing, and integrate CI pipelines for continuous defect prediction.
